Additional Constraints

  • annotation must not occur in the descendants of annotation

Description

The annotation element is a “block” annotation. Block annotations can be used for popups and other “out of line” effects.

An annotation element is associated with another element by using a reference to an xml:id value. The association can go in either direction. An annotation element can use an annotates attribute on itself to point to an xml:id on another element. Or the other element can use an annotations attribute on itself to point to an xml:id on an annotation element. There is no assumption that an annotation element is associated with its parent or any other ancestor element.

The attribute type of annotations and annotates is plain text, not IDREF or IDREFS. That enables modular content files to form associations with elements in other files without generating validation errors.
